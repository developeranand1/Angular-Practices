What is subscribe()?
  • subscribe() is how you start listening to an Observable.
  • It activates the Observable (they are lazy, meaning they do nothing until subscribed).
  • You provide a function that will run whenever a new value is emitted.
What is unsubscribe()?
  • unsubscribe() is used to stop listening to the Observable.
  • This is especially important for long-running Observables like:
    • interval, timer, event listeners, or websocket Observables
    • Component-based Observables (to avoid memory leaks)
Why is it important?

If you don't unsubscribe from infinite or long-lived Observables, they keep running even after the component is destroyed — causing:

  • Memory leaks
  • Unexpected behavior

Methods

  • subscribe() :Start receiving data from an Observable
  • unsubscribe() :Stop receiving data, free resources
When do you need to unsubscribe?
  • Yes: Infinite streams (interval, websocket, event, custom Observables)
  • No: One-time Observables (like HTTP calls) – Angular handles them automatically
