1. What is a Closure in JavaScript?

A closure is a function that has access to its own scope, the outer function's scope, and the global scope even after the outer function has returned. Closures are created every time a function is created in JavaScript.

2. What is the Event Loop in JavaScript?

The Event Loop is part of JavaScript's concurrency model. It allows non-blocking I/O by using a queue and loop. The event loop picks up functions from the callback queue and pushes them into the call stack only when it's empty.

How it works:
  • Call Stack executes code.
  • Web APIs handle async operations (like setTimeout or fetch).
  • Callback Queue stores ready callbacks.
  • Event Loop checks the stack and pushes queued callbacks when it's empty.
3. What is the difference between == and ===?

== is the abstract equality operator that performs type coercion before comparing, while === is the strict equality operator that does not perform type coercion.

4. What are Promises and how do they work?

A Promise is an object representing the eventual completion or failure of an asynchronous operation. It has three states: pending, fulfilled, and rejected.

Bonus: Mention async/await for cleaner syntax.

5. Explain this keyword in JavaScript.

The value of this depends on how a function is invoked. In the global scope, this refers to the window (or global in Node). Inside a method, it refers to the object owning the method.
